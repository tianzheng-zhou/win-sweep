# win-sweep

一个 VS Code Copilot Skill，用于 Windows 系统清理与优化。

> **⚠️ 风险警告**
>
> 本工具通过 AI 辅助修改 Windows 系统配置（服务、启动项、计划任务等）。**系统配置修改具有固有风险**，错误的操作可能导致软件无法运行、系统不稳定甚至无法启动。
>
> - 本工具已内置分级安全机制（见下方），但**你需要自行判断是否接受每一项修改**。
> - 作者对因使用本工具导致的任何系统损坏、数据丢失或其他问题**不承担任何责任**。
> - 建议在操作前创建系统还原点。

## 功能

win-sweep 通过 PowerShell 脚本诊断和清理 Windows 系统膨胀：

- **系统诊断** — 扫描磁盘用量、已安装软件、启动项、服务、计划任务、内存占用排行
- **服务优化** — 基于通用判断框架，将不必要的 Auto 服务改为 Manual/Disabled
- **启动项管理** — 禁用启动项并安全备份到 `RunDisabled` 注册表键
- **计划任务清理** — 禁用不必要的计划任务（特别是绕过服务禁用的遥测任务）
- **遥测完整禁用** — 跨服务 + 计划任务 + 启动项三层排查，防止遥测组件互相拉起
- **可疑服务检测** — 量化风险评分（12 个信号），识别残留、未签名或来源不明的服务

## 安装

复制此文件夹到 Copilot 技能目录：

```
~/.copilot/skills/win-sweep/
```

或克隆到项目中：


## 使用

当你向 Copilot 询问 Windows 系统清理、服务优化、启动项管理或磁盘空间回收时，此技能会自动触发。

典型对话示例：

```
> 我的电脑开机很慢，帮我看看有什么可以优化的
> 扫一下系统里有没有遥测服务
> 帮我检查一下有没有可疑的服务残留
```

## 环境要求

- Windows 10/11
- PowerShell 5.1+
- 无外部依赖

### 管理员权限

win-sweep 的大部分修改操作（服务、计划任务、HKLM 注册表）需要管理员权限。**AI 工具的终端继承父进程的权限**，所以需要在启动 AI 工具之前获取管理员权限。

**各工具的提权方式：**

| 工具类型 | 提权方法 |
|----------|----------|
| **VS Code**（Copilot） | 右键 VS Code 图标 → 以管理员身份运行 |
| **终端类工具**（Claude Code, Codex CLI, OpenCode, Kimi Code, Gemini CLI 等） | 先打开管理员终端，再在其中启动工具 |

**打开管理员终端：**
- **方式 A**：右键开始按钮 → 终端(管理员) / Windows PowerShell(管理员)
- **方式 B**：搜索 `PowerShell` → 右键 → 以管理员身份运行
- **方式 C**：`Win+X` → 选择带"管理员"字样的选项
- **Windows Terminal**：设置 → 默认值 → 以管理员身份运行此配置文件

然后在该终端中正常启动你的 AI 工具即可，工具内的所有命令自动继承管理员权限。

> 如果你不想以管理员身份运行，也可以正常使用——诊断扫描和 HKCU 启动项管理不需要管理员权限。涉及需要提权的操作时，AI 会检测权限不足并提前告知，而非直接执行报错。

| 操作 | 需要管理员 |
|------|------------|
| 系统诊断（diagnose.ps1） | 部分（服务详细信息需要） |
| 服务优化（optimize-services.ps1） | 是 |
| HKCU 启动项管理（manage-startups.ps1） | 否 |
| HKLM 启动项管理（manage-startups.ps1 -Scope HKLM） | 是 |
| 计划任务清理（clean-tasks.ps1） | 是 |
| 可疑服务检测（detect-suspicious.ps1） | 是 |
| 变更验证（verify.ps1） | 部分 |

## 安全机制

### 操作分级确认

不同危险程度的操作有不同的确认要求：

| 等级 | 操作 | 确认方式 |
|------|------|----------|
| 只读 | 诊断、扫描 | 无需确认，随时运行 |
| 低危 | 服务 Auto → Manual | 汇总表一次确认，可部分排除 |
| 中危 | 服务 Auto → Disabled、禁用计划任务 | 汇总表确认 + 标注影响 |
| 高危 | 删除服务、删除注册表项 | 先备份 → 逐项确认 → 展示回滚命令 |

### 其他防护

- **系统还原点** — 首次修改前建议创建还原点
- **可恢复设计** — 启动项移到 `RunDisabled` 备份而非直接删除；服务优先改为 Manual 而非 Disabled
- **备份后删除** — 高危操作前强制 `reg export` 导出注册表，提供回滚命令
- **变更日志** — 每次修改输出时间戳、命令和原值→新值，便于审计

### 设计原则

参考文档提供的是**通用判断框架**（如何评估任意服务），而非硬编码的服务名列表。AI 遇到未知服务时能套用决策流程独立判断，不依赖预定义名单。
